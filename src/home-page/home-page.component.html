Overview

This project focuses on allowing developers to set up infinitely scrolling list pages with minimal setup/frustration.

A number of features are supported out of the box:

  • Infinitely scrolling list
  • Turning columns on/off
  • Reordering columns (active or inactive)
  • Setting column widths as a fraction of a page
  • Setting minimum/maximum widths for a column
  • Setting a column to a fixed width
  • Left/center/right aligning content
  • Adding/removing rows from an existing list
  • Allowing users to resize columns
  • Keep a column always visible on-screen
  • Show/hide columns at different screen widths
  • For data retrieval, choose between a basic array , an asynchronous list , a paginated list , or a custom solution


Documentation

Here, you can find information on features and implementations. These are separated into four broad categories:

  • Main Component
  • Cells
  • Header Cells
  • Rows

Main Component covers any information on inputs or customization applied directly to the base element, <virtual-scroll>, including settings for row size, table height, buffers, et cetera.

Cells covers information relevant to configuring a given column inside of the table, including settings for content alignment, responsiveness, column spacing and enabling/disabling.

Header Cells briefly covers how to provide custom templates for header cells and how to link them to the row cell by name.

Rows covers how to provide custom row or header row templates, for row-level functionality or styling.

On the left is a list of links to live examples for all provided functionality, with copyable code views for each example to facilitate initial setup.

HeightPx

By default, the size of the table displayed will be 500px or the total size of its content, whichever is smaller. Pass a custom heightPx value if you want a static height for your table.
