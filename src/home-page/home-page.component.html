Virtual Scroll

Looking for how to run locally or contribute? Please visit the README.md.



Overview

This project focuses on allowing developers to set up infinitely scrolling list pages with minimal setup/frustration.

A number of features are supported out of the box:

  • Infinitely scrolling list
  • Turning columns on/off
  • Reordering columns (active or inactive)
  • Setting column widths as a fraction of a page
  • Setting minimum/maximum/fixed widths for a column
  • Left/center/right aligning content
  • Adding/removing rows from an existing list
  • Allowing users to resize columns
  • Keep a column always visible on-screen
  • Show/hide columns at different screen widths
  • For data retrieval, choose between a basic array , an asynchronous list , a paginated list , or a custom solution


Documentation

Here, you can find information on features and implementations. These are separated into four broad categories:

  • Main Component
  • Cells
  • Header Cells
  • Rows
  • Miscellaneous

Main Component covers any information on inputs or customization applied directly to the base element, <virtual-scroll>, including settings for row size, table height, buffers, et cetera.

Cells covers information relevant to configuring a given column inside of the table, including settings for content alignment, responsiveness, column spacing and enabling/disabling.

Header Cells briefly covers how to provide custom templates for header cells and how to link them to the row cell by name.

Rows covers how to provide custom row or header row templates, for row-level functionality or styling.

Miscellaneous covers how to handle common list page actions, such as sorting and searching.

HeightPx

By default, the size of the table displayed will be 500px or the total size of its content, whichever is smaller. Pass a custom heightPx value if you want a static height for your table.

Small Content Table
Custom static height table

HeightVh

Table height can also be set as some fraction of the viewport height, from 0-100, by passing a custom heightVh value.
Note: this will override the heightPx value, so if both are passed in, only heightVh will take effect.

Example

Offset

The offset will reduce the height of the table by some pixel value.
This is most commonly used in conjunction with heightVh, so that the table height will be 100% of the viewport height - some pixel value, essentially the same as calc(100vh - 500px).

Example

ItemSize

The height of a row is by default 48px, but can be modified as needed to some other value.
Note: The height of every row must always be uniform, to calculate the size of the scrollbar.

Example

MinRowBuffer

The minimum number of rows to pre-render outside of the current viewport.
Whenever the viewport falls below the minimum number of buffered elements, it'll start rendering to fill the buffer until it hits the maximum.

Example

MaxRowBuffer

The maximum number of rows to pre-render outside of the current viewport.
Whenever the viewport falls below the minimum number of buffered elements, it'll start rendering to fill the buffer until it hits the maximum.

Example

CanResize

This attribute, when applied to the <virtual-scroll> element, will allow the user to resize columns,
Double-clicking on a resize bar will reset the columns to their original sizes.
Note: Whenever any one column is resized, all columns will be moved to fixedWidth, and no longer flex with the screen size.

Example

ResizeWidth

Defines the width of the resize bar displayed between each column in pixels (default 8).
Note: This property is only used when canResize has been applied, otherwise it will have no effect.

Example

Loading

Displays a default loading animation while the developer waits for any necessary actions.
The component won't start building the page until loading goes to false.
Note: The loading state will also display when the dataSource is asynchronously supplied

Loading Example

LoadingTemplate

For the developer to override the default loading animation with a custom implementation.
It'll be displayed in replacement of the default animation, whenever the loading flag is true, or dataSource isn't yet defined, or the size of the list isn't yet populated.

Example

EmptyStateTemplate

For the developer to override the default empty state with a custom implementation.
It'll be displayed in replacement of the default banner, whenever the dataSource is initialized and finished loading, and the list is of length 0.

Example

Cell Padding

The default cell padding of 16px can be overwritten if you want more/less padding for each side, must be provided as a value in pixels.

Cells

Name

The name property for a cell provides both the default implementation for the header, and serves as a unique identifier for that column. When names aren't provided, behavior is undefined, and the table is unlikely to work properly.

Example

Alignment

The content in a column is by default left aligned, but can be center or right aligned as well, by modifying the align property in a cell definition.

Example

Flex Width

By default, all columns are defined as flex: 1, to evenly distribute space between them.
The columns can be modified to take up different portions of the space by playing with the flex property.

Example

MinWidth

When defining a column, we can specify minimum width (in pixels) as a boundary for shrinking with screen size.

Example

MaxWidth

When defining a column, we can specify maximum width (in pixels) as a boundary for growing with screen size.

Example

FixedWidth

When defining a column, we can specify a fixed width (in pixels) to make a column take a static width, without responding to screen size.

Example

IsActive

Columns can be initialized as inactive, hiding them from the user. This can be useful for re-using tables, or hiding extraneous information intially.

Example

IsSticky

When the screen width is too small to fit all the columns on-screen, the column designated as sticky will 'stick' to either side of the viewport.

Example

Context

When defining a cell, we have access to a number of values in the context, such as:

  • The item associated with the cell
  • The index of the row
  • The index of the cell
  • The name of the cell
